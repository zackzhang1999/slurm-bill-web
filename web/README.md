# Slurm Billing Web Application

Slurm计费系统的Web管理界面，支持管理员和普通用户两种角色，提供完整的密码认证和管理功能。

## 功能特性

### 双角色权限系统

#### 管理员
- 使用密码登录（默认：`changeme`）
- 查看所有用户的统计信息
- 管理用户余额（充值、设置信用额度）
- 暂停/激活用户账户
- **管理用户密码（重置为默认密码）**
- 访问系统设置页面
- 查看消费排行

#### 普通用户
- 使用用户名+密码登录
- **默认密码：`123456`**
- **首次登录后强制修改密码**
- 只能查看自己的作业记录和费用
- 查看个人账户余额和充值历史
- 查看个人费用趋势和统计报表
- 无法访问管理功能

### 用户密码管理

#### 密码安全特性
- ✅ 用户密码使用 PBKDF2-SHA256 哈希存储
- ✅ 首次登录强制修改默认密码
- ✅ 管理员可重置用户密码
- ✅ 密码状态显示（默认/已修改）
- ✅ 密码长度至少6位

#### 管理员密码管理功能
- 查看用户密码状态（默认密码警告）
- 一键重置用户密码为 `123456`
- 重置后用户下次登录需修改密码

### 功能模块

#### 仪表盘
- 今日/本月费用实时统计
- 30日费用趋势图表
- 分区费用占比饼图
- 消费排行Top 10（仅管理员）
- 系统总览（总作业数、用户数、余额等）

#### 用户管理（管理员）/ 我的账户（普通用户）
- **管理员**：用户余额列表、搜索筛选、充值、信用额度设置、**密码重置**
- **普通用户**：个人余额、充值记录、作业记录、费用趋势

#### 作业查询
- 多条件筛选（管理员可筛选所有用户，普通用户只看自己）
- 作业详情弹窗展示
- 资源使用统计
- 费用明细查看
- 数据导出功能

#### 统计报表
- 每日统计（趋势图+明细表）
- 账户统计（仅管理员，费用分布+明细表）
- 分区统计（占比+作业数）
- 支持多时间范围选择
- 报表导出功能

#### 系统设置（仅管理员）
- 计费费率查看
- 分区倍率配置
- 折扣配置
- 系统信息展示

## 安装要求

- Python 3.7+
- Flask 3.0+
- Werkzeug（用于密码哈希）
- 现有的Slurm计费系统数据库

## 快速开始

### 1. 安装依赖

```bash
cd web
pip3 install -r requirements.txt
```

### 2. 启动Web服务器

```bash
# 使用启动脚本
bash start.sh

# 或直接启动
python3 app.py
```

### 3. 访问Web界面

打开浏览器访问: http://localhost:5000

## 登录方式

### 普通用户登录
1. 选择"普通用户"标签
2. 输入用户名（如：`amax`, `zack`）
3. 输入密码（首次登录使用默认密码 `123456`）
4. 首次登录后强制修改密码

### 管理员登录
1. 选择"管理员"标签
2. 输入密码：`changeme`
3. 拥有完整管理权限

## 用户密码说明

### 默认密码
- 新用户首次登录默认密码为：`123456`
- 系统会自动识别首次登录，强制要求修改密码

### 密码修改
- **用户自行修改**：登录后可在"我的账户"页面修改
- **管理员重置**：管理员可在"用户管理"页面重置任意用户密码

### 密码存储
- 密码使用 PBKDF2-SHA256 算法哈希存储
- 数据库中不保存明文密码
- 即使是管理员也无法查看用户原始密码

## 配置说明

### Web配置 (config.yaml)

```yaml
web:
  enabled: true
  host: "0.0.0.0"
  port: 8080
  
  auth:
    enabled: true
    admin_password: "changeme"  # 修改此密码
    secret_key: "your-secret-key-here"
```

### 数据库表结构

系统使用以下数据表：
- `job_records` - 作业记录
- `account_balance` - 账户余额
- `recharge_records` - 充值记录
- `user_passwords` - **用户密码（新增）**

### 初始化用户密码

系统启动时会自动：
1. 创建 `user_passwords` 表（如果不存在）
2. 为新用户初始化默认密码 `123456`

## API接口

### 用户认证
- `POST /login` - 登录（支持admin和user两种类型）
- `GET /api/user/info` - 获取当前用户信息
- `POST /change-password` - 修改密码（普通用户）

### 密码管理（仅管理员）
- `POST /api/user/<username>/password` - 重置用户密码
- `GET /api/user/<username>/password-status` - 获取密码状态

## 安全说明

### 修改默认管理员密码
```yaml
# /etc/slurm-bill/config.yaml
web:
  auth:
    admin_password: "YourStrongPassword123!"
```

### 用户密码安全
1. **强制修改默认密码** - 首次登录必须修改
2. **密码哈希存储** - 使用PBKDF2-SHA256
3. **密码长度限制** - 至少6位
4. **密码状态显示** - 管理员可查看谁在使用默认密码

### 生产环境建议
1. 使用HTTPS
2. 配置Nginx反向代理
3. 限制访问IP范围
4. 定期更换管理员密码

## 故障排查

### 用户无法登录
1. 检查用户名是否存在作业记录
2. 检查密码是否正确（区分大小写）
3. 如果是新用户，检查是否已初始化密码表

### 重置用户密码
管理员可在"用户管理"页面，点击用户详情中的"重置密码"按钮。

### 检查密码表
```bash
sqlite3 /var/lib/slurm-bill/billing.db "SELECT user, is_default FROM user_passwords;"
```
